# 项目进度

更新时间：2026-04-15

## 当前定位

当前项目已经进入“真实纳管 MVP + 单机控制面”阶段，不再是纯 mock。

本轮新增进展：

- 项目进度文档已独立沉淀，后续按模块持续更新
- 控制面鉴权第一版已落地，页面与 operator API 已启用 session cookie 登录保护
- 任务中心已升级到第一版可操作态，支持筛选、摘要和直接重试/复探
- 任务中心已打通关联执行回显，可直接跳到指定 operation 的运维终端
- SSH 传输层已拆出非交互执行通道，避免初始化/批量脚本误走 Web Shell 的交互式 TTY
- 统一用户/协议模板/发布中心第一版已落地，前后端已接通 `AccessUser / ProxyProfile / NodeGroup / ConfigRelease`
- 持续巡检第一版已落地：控制面可按计划自动创建 `scheduled_probe` 任务并回写节点健康状态
- 总览、任务中心、节点清单 hover 已接入自动巡检状态，可直接看到最近一轮、下一轮和周期任务沉淀
- 系统用户模块已落地并通过真实节点验收，支持 Linux 账号、SSH 公钥、sudo 与批量下发
- 系统模板模块已落地并通过真实节点验收，支持沉淀初始化/基线脚本并批量下发到直连或中转节点
- 节点详情页已接入“模板动作”区，可直接对当前节点执行初始化模板或系统模板下发
- 节点详情 -> 系统模板中心已支持带当前节点与模板的深链预填

已经跑通的主链路：

- 未登录访问控制面会自动跳转登录页，登录后按 `next` 回到原页面
- 新节点执行 bootstrap 后自动注册到平台
- 服务端完成节点去重、事实归一化、任务落库
- 平台可回传 SSH 公钥，节点可回调 bootstrap 完成
- 控制面可触发初始化、自动首探、手动复探
- 前端已接真实节点、任务、探测、令牌、操作数据
- 系统用户与系统模板下发记录已接入真实终端回显链路

当前仍未完成的核心目标：

- 持续巡检告警、趋势与策略联动
- 自愈与自动下线/替换
- 厂商 API 自动建机
- 面板/NMS 与代理服务真实接入
- 数据库化与多实例化

## 当前主要风险

- JSON 文件持久化只适合单机 MVP
- 控制面会话目前仍是单进程内存 session，服务重启后会失效，也不支持多实例共享
- 初始化模板虽然已切回非交互执行通道，但仍需继续验证超时、幂等和异常重试细节
- Web Shell 仍是轮询式会话，不是生产级 bastion
- 任务系统还缺完整动作和调度能力
- 厂商模块还没进入真实资源管理阶段
